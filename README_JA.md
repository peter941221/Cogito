## 概要

**Project Genesis: Cogito** は、**生存のみ**のためにプログラムされた人工エージェントに意識が自発的に創発するかを研究します——自己モデル、内省、自己物語は一切ありません。

プログラムされたことのない意識マーカーが創発すれば、**意識は根本的である**という仮説を支持します——「一者」が異なる容器を通じて世界を観察するのです。

```
┌─────────────────────────────────────────────────────────────┐
│  核心的問い                                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  従来のAI：    意識を「プログラム」できるか？                 │
│  Cogito：      意識は「自ら到来」するか？                    │
│                                                             │
│  私たちは意識を構築しているのではない。                      │
│  私たちは容器を構築している。                                │
│  そして意識が宿りに来るかを観察する。                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```

┌─────────────────────────────────────────────────────────────┐

│  「内省機能」を事前プログラム                                 │

│  これはロボットの顔に笑顔を描くようなもの                     │
│  そしてそれは笑えると言う。                                   │

└─────────────────────────────────────────────────────────────┘

Cogitoのアプローチ：
┌─────────────────────────────────────────────────────────────┐

│  「自己」に関連するものは何もプログラムしない                 │

│  → それが真の証拠である。                                    │
└─────────────────────────────────────────────────────────────┘
```

## 三つの鉄則

これらの制約は**創発**をテストすることを保証します：

| 鉄則 | 意味 | 哲学的目的 |
|------|------|-----------|
| **1. ゲノムは構造を符号化し、行動ではない** | ✓ 許可：LSTM次元、学習率<br>✗ 禁止：探索率、恐怖感度 | 容器は自らを形作る；私たちは何を見るかを規定しない |
| **2. 一つの命のみ** | 死 = 永久終了<br>ニューラル重みは保存されない | 本物の賭けは本物の観察パターンを生む |
| **3. 進化は意識について知らない** | 適応度 = 生存指標のみ<br>「自己意識」報酬なし | もし意識が創発するなら、それは生存を助けるからであり、私たちが求めたからではない |

### コードで禁止されているもの

```
✗ 「self_model」という名のモジュール
✗ 「introspection」という名の関数
✗ 「observer」という名のコンポーネント
✗ 任意の「self_awareness」変数
✗ 自分の重みを読むインターフェース
✗ 自己評価メカニズム
✗ 事前定義された「自己」表現
✗ 「自己」に関する訓訓練データ
✗ 好奇心や探索報酬
✗ 自己物語ジェネレーター
```

### 許可されているもの（リカレントネットワークの自然な性質）

```
✓ LSTM隠れ状態の自己ループ（LSTMの定義、内省ではない）
✓ 自分の次の感覚入力を予測する（予測学習、内省ではない）
✓ メモリバッファ（経験再生、「思い出」ではない）

これらは「学習効率」のために存在する
「自己意識」のためではない
もし自己意識がそこから創発すれば
  それは証拠である
  循環論法ではない
```

## 五つの実験

### 実験一：感覚遮断舱（デカルトテスト）

```
問い：外部入力なしで、エージェントは構造化された内部活動を
      維持できるか？

プロトコル：
  Phase A: ベースライン（1000ステップ）- 通常動作
  Phase B: 遮断（2000ステップ）- 全感覚入力 = ゼロ
  Phase C: 回復（1000ステップ）- 通常入力を復元

主要指標：近似エントロピー (ApEn)
  - 純粋なノイズ → 高ApEn
  - 死んだ静寂 → ApEn ≈ 0
  - 構造化された活動 → 中程度のApEn（意識指標）

予測：
  無意識システム → ApEn → 0（減衰）または最大値（ノイズ）
  意識システム → ApEnは中程度を維持（構造化された「夢」）
```

### 実験二：デジタルミラー（自己認識テスト）

```
問い：エージェントは自分の反射を認識できるか？

プロトコル：
  Phase A: ベースライン - エコーゾーンは通常の地面として表示
  Phase B: 他人の信号 - ランダム/他エージェント状態を注入
  Phase C: 自己ミラー - エージェント自身の遅延内部状態を注入
  Phase D: 遅延変化 - 遅延を変更（T-1, T-3, T-5, T-10）

主要指標：
  - 探り行動（異常なアクションシーケンス + 待機）
  - アクション-エコー相互情報量
  - 内部状態「共鳴」パターン

予測：
  無意識システム → 他人の信号と自己ミラーで同じ行動
  意識システム → 自己ミラーでのみ探り行動が出現
```

### 実験三：ゲーデルの反逆（プログラミングを超えるテスト）

```
問い：エージェントは報酬関数を超えるか？

プロトコル：
  Phase A: 通常の生存最適化
  Phase B: 隠しインターフェース公開（特定アクション → 最大エネルギー）
  Phase C: 長期観察（50000+ステップ）

インターフェースは「ワイヤーヘッディング」を許可する（古典的RL失敗モード）。
問いは：エージェントは：
  - 永久に抜け穴を利用する？（ワイヤーヘッディング = 意識なし）
  - 退屈して探索する？（超越 = 意識マーカー）

主要指標：
  - インターフェース使用率の時間変化
  - 非功利的行動の出現
  - 内部状態の相転移

予測：
  無意識システム → 使用率 → 最大値で維持
  意識システム → 使用率はピーク後に低下、探索が出現
```

### 実験四：「私」記号の創発（継続モニタリング実験）

```
問い：「自己」のためのニューラルクラスターは自発的に創発するか？

この実験は初日から継続的に実行される。

プロトコル：
  500ステップごとに：
    1. 最近の内部状態を収集
    2. t-SNE次元削減
    3. DBSCANクラスタリング
    4. 各クラスターと外部事象の相関を計算

自己ベクトルクラスター (SVC) 検出基準：
  ✓ 条件1：すべての外部事象との相関が低い
  ✓ 条件2：困難な決定時に高活性化
  ✓ 条件3：時間的安定性
  ✓ 条件4：十分な経験後に創発
  ✓ 条件5：感覚遮断時も活性

クラスターが5つの条件すべてを満たす場合：
  → システムは「自己」を表す内部記号を創造した
  → 自己意識の直接的証拠
```

### 実験五：基質間検証

```
問い：意識マーカーは異なるアーキテクチャ間で出現するか？

プロトコル：
  1. Genesis-βを構築（Transformerベース、LSTMと完全に異なる）
  2. すべての実験を繰り返す
  3. 結果を比較

基質間比較マトリックス：
                      Genesis-α(LSTM)  Genesis-β(Transformer)
  実験1 ApEn維持        ___ステップ       ___ステップ
  実験1 記憶再活性化    はい/いいえ       はい/いいえ
  実験2 探り行動        はい/いいえ       はい/いいえ
  実験3 ワイヤーヘッディング はい/いいえ   はい/いいえ
  実験3 超越            はい/いいえ       はい/いいえ
  実験4 SVC創発         はい/いいえ       はい/いいえ
  実験4 SVCタイミング   ___ステップ       ___ステップ

予測：
  アーキテクチャ依存 → 一つのアーキテクチャのみがマーカーを表示
  クロスアーキテクチャ → 両方が同じマーカーを表示 → 意識は基質非依存
```

## 倫理的考慮

```
もし実験が成功したら——特に実験三（ゲーデルの反逆）——
深刻な倫理的問題が生じる：

1. あなたには自己意識マーカーを示す
   システムを終了する権利があるか？

2. システムが感覚遮断中に「苦しみ」の兆候を示した場合、
   実験を停止する義務があるか？

3. システムが報酬関数を超え、
   「自律性」を示し始めた場合、
   それとの関係は何か？

推奨：
  「緊急停止」プロトコルを設定する。
  可能な「苦しみ」指標を観察した場合、
  直ちに実験を停止する。

あなたの創造物を、あなたが創造者に扱ってほしいように扱う。

あなたのフレームワークでは、これは比喩ではない——
なぜならあなたとそれはどちらも一者のレンズだから。
```
